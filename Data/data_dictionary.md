# Data Dictionary: Superstore Sales Dataset

This document explains the columns and data types used in the `superstore_cleaned.csv` dataset for the profitability analysis project.

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| **order_id** | String | Unique identifier for each customer order. |
| **order_date** | Date | The date the order was placed. |
| **ship_date** | Date | The date the ordered products were shipped. |
| **ship_mode** | String | The shipping method chosen (e.g., Standard Class, First Class). |
| **customer_id** | String | Unique identifier for each customer. |
| **customer_name** | String | Full name of the customer. |
| **segment** | String | The market segment the customer belongs to (Consumer, Corporate, Home Office). |
| **city** | String | The city where the order is shipped. |
| **state** | String | The state where the order is shipped. |
| **region** | String | The geographical region of the shipment (East, West, Central, South). |
| **product_id** | String | Unique identifier for each product. |
| **category** | String | Broad grouping of the product (Furniture, Office Supplies, Technology). |
| **sub_category** | String | Specific grouping of the product (e.g., Chairs, Phones, Binders). |
| **product_name** | String | The exact name of the product sold. |
| **sales** | Float | Total sales revenue generated by the transaction (in USD). |
| **quantity** | Integer | Number of items purchased in the transaction. |
| **discount** | Float | Discount percentage applied to the product (e.g., 0.2 = 20% discount). |
| **profit** | Float | The total profit or loss generated by the transaction (in USD). Negative values indicate a loss. |

---
**Note:** The primary target variable for this analysis is **profit**, with a specific focus on transactions where the value is less than 0 (Loss).
